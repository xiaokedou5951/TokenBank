# TokenBank 部署指南

## 1. 目标

说明合约和前端的部署流程、环境变量配置及部署检查清单。

## 2. 关键结论

- 合约部署使用 Foundry Script（`Deploy.s.sol`），依次部署 MyTokenPermit → TokenBankPermit。
- 前端为纯静态部署（Next.js），合约地址通过环境变量注入。
- 本地开发使用 Anvil 默认私钥和地址；测试网/主网需配置真实私钥和 RPC。

## 3. 合约部署

### 3.1 部署脚本

部署脚本位于 `contracts/script/Deploy.s.sol`，执行流程：

```
1. 读取 PRIVATE_KEY 和 INITIAL_SUPPLY 环境变量
2. 部署 MyTokenPermit(initialSupply) → 代币合约
3. 部署 TokenBankPermit(address(myTokenPermit)) → 银行合约
4. 输出两个合约地址
```

### 3.2 配置环境变量

在 `contracts/` 目录创建 `.env` 文件：

```bash
cp .env.example .env
```

编辑 `.env`：

| 变量 | 说明 | 示例 |
|------|------|------|
| `PRIVATE_KEY` | 部署者私钥（带 0x 前缀） | Anvil 默认: `0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80` |
| `INITIAL_SUPPLY` | 代币初始供应量（不含精度） | `1000000000`（即 10 亿枚代币） |
| `SEPOLIA_RPC_URL` | Sepolia RPC 地址 | `https://eth-sepolia.g.alchemy.com/v2/YOUR_KEY` |
| `ETHERSCAN_API_KEY` | Etherscan API Key（合约验证用） | 从 etherscan.io 获取 |

### 3.3 本地部署（Anvil）

```bash
# 1. 启动 Anvil
anvil

# 2. 部署合约（新终端）
cd contracts
forge script script/Deploy.s.sol \
  --rpc-url http://127.0.0.1:8545 \
  --broadcast
```

部署成功后，终端会输出合约地址：

```
MyTokenPermit deployed to: 0x5FbDB2315678afecb367f032d93F642f64180aa3
TokenBankPermit deployed to: 0xe7f1725E7734CE288F8367e1Bb143E90bb3F0512
```

### 3.4 Sepolia 测试网部署

```bash
cd contracts
forge script script/Deploy.s.sol \
  --rpc-url sepolia \
  --broadcast \
  --verify
```

> **注意**：Sepolia 部署需要真实的 ETH 作为 gas，确保部署账户有足够余额。

### 3.5 合约验证

部署时自动验证（`--verify`），或手动验证：

```bash
forge verify-contract <CONTRACT_ADDRESS> <CONTRACT_NAME> --chain sepolia
```

## 4. 前端部署

### 4.1 配置环境变量

在 `frontend/` 目录创建 `.env.local`：

```bash
cp .env.local.example .env.local
```

根据部署网络填入实际值：

| 变量 | 本地开发 | Sepolia |
|------|----------|---------|
| `NEXT_PUBLIC_TOKEN_ADDRESS` | 部署输出的 MyTokenPermit 地址 | 部署输出的 MyTokenPermit 地址 |
| `NEXT_PUBLIC_TOKENBANK_ADDRESS` | 部署输出的 TokenBankPermit 地址 | 部署输出的 TokenBankPermit 地址 |
| `NEXT_PUBLIC_RPC_URL` | `http://127.0.0.1:8545` | `https://eth-sepolia.g.alchemy.com/v2/YOUR_KEY` |
| `NEXT_PUBLIC_CHAIN_ID` | `31337` | `11155111` |

### 4.2 本地开发

```bash
cd frontend
npm install
npm run dev
```

### 4.3 生产构建

```bash
cd frontend
npm run build
npm run start
```

### 4.4 部署到 Vercel（推荐）

1. 将项目推送到 GitHub
2. 在 Vercel 导入项目，设置 Root Directory 为 `frontend`
3. 配置环境变量（同 4.1）
4. 部署

## 5. 部署检查清单

### 5.1 合约部署前

## 6. 常见问题

### 6.1 部署脚本报 "insufficient funds"

确认部署账户在目标网络有足够 ETH。本地 Anvil 默认账户已有余额。

### 6.2 合约验证失败

- 确认 `ETHERSCAN_API_KEY` 已正确配置
- 确认编译器版本匹配（`foundry.toml` 中 `solc_version = "0.8.30"`）
- 可使用 `--verifier-url` 指定验证 API

### 6.3 前端连接合约失败

- 确认合约地址与部署地址一致
- 确认 RPC URL 可访问
- 确认 Chain ID 与网络匹配
- 浏览器控制台查看是否有 RPC 报错

### 6.4 Anvil 重启后前端数据异常

Anvil 重启后链状态重置，但前端 React Query 缓存未失效。解决方式：

- 刷新页面
- MetaMask 中重置账户 nonce
