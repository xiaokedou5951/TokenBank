# TokenBank 环境搭建

## 1. 目标

说明本地开发环境的准备步骤，包括依赖安装、合约编译测试、前端启动。

## 2. 关键结论

- 项目分为 `contracts/`（Foundry）和 `frontend/`（Next.js）两个独立子项目。
- 本地开发需要 Anvil（Foundry 内置）作为本地链，MetaMask 或其他钱包连接到 `localhost:8545`。
- 前端依赖 Node.js 18+，合约依赖 Foundry 工具链。

## 3. 前置条件

| 工具 | 版本要求 | 用途 |
|------|----------|------|
| Foundry (forge/cast/anvil) | 最新稳定版 | 合约编译、测试、部署 |
| Node.js | 18+ | 前端运行 |
| npm | 9+ | 前端依赖管理 |
| MetaMask 或兼容钱包 | - | 连接本地链 |

### 3.1 安装 Foundry

```bash
curl -L https://foundry.paradigm.xyz | bash
foundryup
```

### 3.2 安装 Node.js

推荐使用 nvm 管理 Node.js 版本：

```bash
nvm install 18
nvm use 18
```

## 4. 合约环境搭建

### 4.1 安装依赖

```bash
cd contracts
forge install
```

这将安装 `forge-std`、`openzeppelin-contracts` 等依赖（通过 git submodule）。

### 4.2 编译合约

```bash
forge build
```

编译输出在 `contracts/out/` 目录。

### 4.3 运行测试

```bash
forge test
```

详细输出：

```bash
forge test -vvv
```

### 4.4 启动本地链

```bash
anvil
```

默认监听 `http://127.0.0.1:8545`，chainId 为 31337。

Anvil 启动时会输出 10 个测试账户及其私钥，可用于开发调试。

## 5. 前端环境搭建

### 5.1 安装依赖

```bash
cd frontend
npm install
```

### 5.2 配置环境变量

复制示例文件并按需修改：

```bash
cp .env.local.example .env.local
```

`.env.local` 内容说明：

| 变量 | 说明 | 默认值（Anvil） |
|------|------|-----------------|
| `NEXT_PUBLIC_TOKEN_ADDRESS` | MyTokenPermit 合约地址 | `0x5FbDB2315678afecb367f032d93F642f64180aa3` |
| `NEXT_PUBLIC_TOKENBANK_ADDRESS` | TokenBankPermit 合约地址 | `0xe7f1725E7734CE288F8367e1Bb143E90bb3F0512` |
| `NEXT_PUBLIC_RPC_URL` | RPC 节点地址 | `http://127.0.0.1:8545` |
| `NEXT_PUBLIC_CHAIN_ID` | 链 ID | `31337` |

> 部署合约后需更新合约地址为实际部署地址。

### 5.3 启动开发服务器

```bash
npm run dev
```

使用 Turbopack 加速开发，默认监听 `http://localhost:3000`。

## 6. 钱包配置

### 6.1 MetaMask 连接 Anvil

1. 打开 MetaMask → 添加网络
2. 配置如下：

| 字段 | 值 |
|------|-----|
| 网络名称 | Anvil Local |
| RPC URL | `http://127.0.0.1:8545` |
| 链 ID | `31337` |
| 货币符号 | ETH |

3. 导入 Anvil 测试账户：使用 Anvil 启动时输出的私钥导入账户。

> **注意**：Anvil 重启后状态会重置，MetaMask 可能因 nonce 不匹配报错，需在 MetaMask 中重置账户（Settings → Advanced → Clear activity tab data）。

## 7. 完整开发流程

```bash
# 终端 1：启动本地链
cd contracts && anvil

# 终端 2：部署合约
cd contracts
forge script script/Deploy.s.sol --rpc-url http://127.0.0.1:8545 --broadcast

# 终端 3：启动前端
cd frontend && npm run dev
```

1. 浏览器打开 `http://localhost:3000`
2. 连接 MetaMask（选择 Anvil Local 网络）
3. 进行存取款操作

## 8. 常见问题

### 8.1 `forge install` 失败

确保已安装 git，且 `contracts/` 目录下有 `.gitmodules`。若 submodule 未初始化：

```bash
git submodule update --init --recursive
```

### 8.2 前端连接不上合约

- 确认 Anvil 正在运行
- 确认 `.env.local` 中合约地址与部署地址一致
- 确认 MetaMask 连接到 Anvil 网络（chainId 31337）

### 8.3 MetaMask 交易报 nonce 错误

Anvil 重启后链状态重置，需在 MetaMask 中重置账户：Settings → Advanced → Clear activity tab data。
