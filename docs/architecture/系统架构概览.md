# TokenBank 系统架构概览

## 1. 目标

描述 TokenBank 的整体系统分层、模块边界、部署拓扑以及主要数据流，帮助读者建立全局视图。

## 2. 关键结论

- TokenBank 采用经典的两层 DApp 架构：**链上合约层** + **浏览器前端层**。
- 合约层包含 `MyTokenPermit`（支持 EIP-2612 的 ERC20 代币）和 `TokenBankPermit`（支持 Permit 存款的银行合约），无管理权限。
- 前端为纯客户端渲染（`'use client'`），通过 RainbowKit 连接钱包，通过 Wagmi Hooks 读写链上数据。
- 前端支持两种存款模式：传统 Approve+Deposit（两步交易）和 Permit Deposit（EIP-2612 签名，一步完成）。
- 所有业务状态均来自链上查询，前端不依赖任何自建后端服务。

## 3. 系统分层

```
┌─────────────────────────────────────────────────────────────┐
│                        浏览器前端层                          │
│  Next.js 16 (App Router) + React 19 + Tailwind CSS 4       │
│  RainbowKit（钱包连接）→ Wagmi / Viem → 合约调用             │
│  存款方式：Approve+Deposit（传统） / Permit Deposit（EIP-2612）│
└───────────────────────────┬─────────────────────────────────┘
                            │ JSON-RPC / Wallet Provider
┌───────────────────────────▼─────────────────────────────────┐
│                        链上合约层                            │
│  MyTokenPermit.sol（ERC20 + EIP-2612 Permit 代币）          │
│  TokenBankPermit.sol（代币存取合约，支持 permitDeposit）     │
└─────────────────────────────────────────────────────────────┘
```

## 4. 模块边界

| 模块 | 职责 | 边界说明 |
|------|------|----------|
| MyTokenPermit | ERC20 + EIP-2612 代币实现 | 负责余额、转账、授权（approve/permit）、nonce 管理 |
| TokenBankPermit | 代币存取托管 | 持有用户存入的代币，记录存款余额，支持传统存款和 Permit 存款 |
| 前端 Hooks (`useToken` / `useTokenBank`) | 封装合约调用 | 不直接操作 UI，只暴露数据和操作函数 |
| 前端组件 | UI 渲染与用户交互 | 不直接调用合约，依赖 Hooks 提供的数据和行为 |
| Wagmi / RainbowKit | 钱包连接与链交互 | 屏蔽不同钱包和链的差异 |

## 5. 部署视图

当前支持三种运行环境：

| 环境 | 链 | RPC | 用途 |
|------|-----|-----|------|
| 本地开发 | Anvil（chainId 31337） | `http://127.0.0.1:8545` | 本地联调 |
| 测试网 | Sepolia | Ankr RPC | 测试验证 |
| 主网 | Mainnet | Ankr RPC | 生产访问（配置中预留） |

合约地址通过前端环境变量注入；本地开发使用 Anvil 默认地址作为 fallback。

部署脚本 `Deploy.s.sol` 依次部署 `MyTokenPermit` → `TokenBankPermit`，并将代币地址注入银行合约。

## 6. 核心数据流

### 6.1 读取余额

```
用户打开页面 → BalanceCard 挂载
                    ↓
            useTokenBalance / useDepositBalance / useTokenAllowance
                    ↓
            Wagmi useReadContract → JSON-RPC → 合约 view 函数
                    ↓
            React Query 缓存 → UI 渲染
```

### 6.2 传统存款流程（Approve + Deposit）

```
用户输入金额 → DepositForm 调用 useApprove().approve()
                    ↓
            钱包签名授权 TokenBankPermit 花费代币
                    ↓
            用户再调用 useDeposit().deposit()
                    ↓
            钱包签名 → TokenBankPermit.deposit() → transferFrom → 更新 balances
                    ↓
            交易确认后 invalidateQueries → 刷新余额
```

### 6.3 Permit 存款流程（一键签名存款）

```
用户输入金额 → DepositForm 调用 handlePermitDeposit()
                    ↓
            读取 nonce → 构建 EIP-712 typed data → signTypedDataAsync()
                    ↓
            钱包弹出 EIP-712 签名请求 → 用户签名
                    ↓
            从签名提取 v/r/s → permitDeposit(amount, deadline, v, r, s)
                    ↓
            TokenBankPermit.permitDeposit()
              → permit() 验证签名并更新 allowance
              → safeTransferFrom() 转入代币
              → emit PermitDeposit
                    ↓
            交易确认后 invalidateQueries → 刷新余额
```
