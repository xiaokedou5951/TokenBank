# TokenBank 前端架构

## 1. 目标

说明 TokenBank 前端的目录结构、核心模块职责、数据流以及关键设计决策。

## 2. 关键结论

- 前端基于 **Next.js 16 App Router + React 19 + Tailwind CSS 4** 构建，开发服务器使用 Turbopack。
- Web3 交互统一通过 **Wagmi v2 + Viem v2** 完成，钱包连接由 **RainbowKit** 提供。
- 状态管理以 **TanStack Query（React Query）** 为核心，Wagmi Hooks 已内置缓存与刷新能力。
- 合约 ABI 采用**内联 `as const` 片段**管理，避免额外生成步骤，仅包含前端实际使用的函数、事件和错误。

## 3. 目录结构

```

│   │   ├── providers.tsx       # RainbowKit / Wagmi / QueryClient Provider 组合
│   │   └── globals.css         # 全局样式与 Tailwind v4 CSS 变量主题
│   ├── components/
│   │   └── web3/
│   │       ├── ConnectWallet.tsx   # 钱包连接按钮
│   │       ├── BalanceCard.tsx     # 聚合显示 Token 余额、存款余额、授权额度
│   │       ├── DepositForm.tsx     # 存款表单（含授权提示）
│   │       ├── WithdrawForm.tsx    # 取款表单
│   │       └── ActivityLog.tsx     # 交易活动 / Receipt 列表
│   ├── hooks/

```

## 4. 核心模块职责

### 4.1 配置层

| 文件 | 职责 |
|------|------|
| `lib/wagmi.ts` | 定义本地 Anvil 链，配置 RainbowKit + Wagmi 支持的链与 RPC transport |
| `lib/contracts.ts` | 定义 `myTokenAbi` / `tokenBankAbi` 内联 ABI 片段，管理合约地址（环境变量优先，Anvil 默认地址 fallback） |
| `app/providers.tsx` | 组合 RainbowKit、WagmiConfig、QueryClientProvider |

### 4.2 Hooks 层

| Hook | 说明 |
|------|------|
| `useTokenBalance(address)` | 读取用户钱包中的 MyToken 余额 |
| `useTokenAllowance(address)` | 读取用户对 TokenBank 的授权额度 |
| `useApprove()` | 调用 `approve(TokenBank, amount)`，交易成功后刷新相关查询 |
| `useDepositBalance(address)` | 读取用户在 TokenBank 中的存款余额 |
| `useDeposit()` | 调用 `TokenBank.deposit(amount)` |
| `useWithdraw()` | 调用 `TokenBank.withdraw(amount)` |

所有写操作 Hook 统一返回 `isPending`（等待签名）、`isConfirming`（等待上链）、`isSuccess`（成功）、`error`（错误）状态，供 UI 展示。

### 4.3 组件层

| 组件 | 职责 |
|------|------|
| `ConnectWallet` | 触发 RainbowKit 连接弹窗 |
| `BalanceCard` | 聚合展示三类余额：钱包 Token 余额、Bank 存款余额、对 Bank 的授权额度 |
| `DepositForm` | 输入金额，先授权（如需要）再存款 |
| `WithdrawForm` | 输入金额，从 Bank 取款 |
| `ActivityLog` | 展示近期交易 hash、状态、成功/失败信息 |

页面布局采用响应式网格：左侧 8/12 为主内容（余额 + 表单），右侧 4/12 为交易记录。

## 5. 数据流

```
用户操作 → React 组件 → 自定义 Hook → Wagmi / Viem → JSON-RPC → 智能合约
                ↑                                    │
                └──── React Query 缓存与刷新 ←────────┘
```

- **读数据**：`useReadContract` 配合 `enabled: !!address`，未连接钱包时不发起请求。
- **写数据**：`useWriteContract` 发起交易，`useWaitForTransactionReceipt` 监听回执。
- **刷新**：交易成功后调用 `queryClient.invalidateQueries({ queryKey: ['readContract'] })`，统一刷新余额与授权。

## 6. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| ABI 管理方式 | 内联 `as const` 片段 | 直接为 Wagmi/Viem 提供类型推断，无需额外类型生成步骤 |
| 合约地址管理 | 环境变量 + Anvil fallback | 支持多环境，本地开发可零配置启动 |
| 状态管理 | React Query（Wagmi 内置） | 无需 Redux 等全局状态库，缓存、重试、刷新已覆盖需求 |
| 组件粒度 | BalanceCard 聚合三类余额 | 减少组件数量，当前页面信息密度低，聚合展示更直观 |
| 链配置 | Anvil + Sepolia + Mainnet | 本地开发、测试网、生产预留 |
